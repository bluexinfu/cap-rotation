## [Unreleased]

> 已規劃或進行中、尚未發版的改動。發版時把本區塊改名為新版本號 + 日期。
> 目前待辦見 [docs/ROADMAP.md](docs/ROADMAP.md)（R6 起）。

### Added
- **資料更新機制說明彈窗**：工具列 🔄 旁新增 ⓘ 鈕，點開說明「資料每交易日（週一～五）台灣時間 18:30 盤後自動更新（GitHub Actions，全自動零人工）；🔄 為重新載入伺服器上已發布的最新資料、**並非即時向交易所重新計算**，若當日更新尚未完成、重載也不會有新資料」。沿用既有 modal（手機亦可點開）。「最新資料」標籤前加綠點示意運作中；🔄 的 tooltip 由「重新整理資料」改為更精確的「重新載入最新資料」。

### Changed
- **四象限中性門檻改用各板塊自身波動 σ**：中性灰的判定由固定 ±3 億改為「各板塊自身 1 個標準差σ」，金額與相對強度兩座標模式共用同一規則、判定一致。消除大小板塊偏誤（大板塊不再幾乎永遠亮、小板塊不再永遠灰），畫面更穩（以 70 個交易日實測：分類翻轉率 14.4%→6.6%、各板塊中性% 離散度 36→7）。同步更新說明面板 ② 的白話解說。（F-V1、F-V2；見 [ADR-0008](docs/decisions/ADR-0008-neutral-threshold-sigma.md)）

### Fixed
- **手機版四象限排版**：四顆象限統計改為等寬並排一行（原本會 3+1 換行），「四象限/點一下篩選」說明小字移到按鈕下方置中；窄螢幕隱藏方位小字省空間。

### Docs
- **新手指南補充「台積電為何獨立」**：② 設計故事第 4 點補上——台積電（2330）獨立成單獨一顆泡泡、不併入任何主題，因其資金量級獨大又橫跨多個主題（CoWoS、先進製程…），併入會讓整組變成「只在看台積電」、蓋掉其他成員輪動（見 [ADR-0006](docs/decisions/ADR-0006-ai-theme-dimension.md)）。
- **K 線區說明釐清兩種模式的選股邏輯**：證交所產業模式配「單一龍頭股」（區間成交金額最大者，因其最具代表性、流動性最好、量價最能反映板塊氛圍）；AI 主題模式則顯示「前三大成員」個股（非單挑一檔）。同時補上挑龍頭股的理由。

## [0.3.0] - 2026-06-18

大版本：新增「AI 主題」分群維度與成員 K 線＋資金流疊加、加權指數大盤基準卡、大盤潮汐與逆勢強勢高亮、上櫃資料源（OHLC 改用 FinMind）、Y 軸 EMA 平滑（R7），以及整體介面改版。

### Added
- **新增「AI 主題」分群維度（可切換）**：右上加「證交所產業 ↔ AI 主題」切換。AI 主題用同一批個股資料、按 10 個 AI 供應鏈子題（+台積電獨立）重新彙總（成員定義見 [docs/ai-themes.md](docs/ai-themes.md)）；點主題泡泡可在右側看該組**成員個股 K 線，下方疊加該股法人資金流（每日淨買長條 + 區間累計線）**，與時間軸同步播放，呈現「籌碼 vs 價格」對照。含量級偏大個股的主題（如電源含台達電）顯示備註提示。指標數學（EMA 平滑、相對強度σ）兩維度共用。屬「觀察用分類、非選股建議」，設計與界線見 [ADR-0006](docs/decisions/ADR-0006-ai-theme-dimension.md)。
- **資料管線新增上櫃資料源**：AI 供應鏈有多檔關鍵股為上櫃（群聯、旺矽、精測、雙鴻、威剛、聯亞）。法人買賣超（股數）取自**櫃買中心官方**端點；**個股歷史 OHLC 與每日收盤改用 FinMind 公開資料 API**——因櫃買官方的收盤端點只回最新一日、不開放歷史（會導致每天 OHLC 相同、且資金流金額用錯收盤價）。修正後上櫃成員 K 線與資金流皆為正確的每日歷史。`data.json` schema 向後相容擴充（板塊新增 `dim`/`members`/`note`）。
- **加權指數（大盤）基準卡（可切換）**：K 線區「＋ 選擇」左側新增「加權指數」切換鈕，開啟後於頂端固定一張大盤卡：加權指數蠟燭 K 線 ＋ 全市場三大法人買賣超（每日長條＋區間累計線）。作為所有主題/個股觀察的市場基準對照。資料源：MI_5MINS_HIST（指數 OHLC）＋ BFI82U（三大法人合計）。`data.json` 新增 `market` 物件。
- **大盤潮汐 + 逆勢強勢高亮（可切換，預設開）**：摘要列顯示「大盤潮汐 漲潮/退潮 ±X 億/天」（全市場三大法人近5日日均）；當大盤退潮時，把「自身仍淨流入(x>0)且力道沒減速(y>=0)」的板塊/主題中、**相對強度(÷σ)最強的前 5 名**用金色高亮並列名——浮現「退潮中最逆勢、最強勢」的標的。兩維度通用，屬觀察用高亮（非買賣訊號）。設計與盲點處理見 [ADR-0007](docs/decisions/ADR-0007-counter-tide-strength.md)。

### Changed
- **介面改版**：標題改為「資金動向觀測」（副標「台股板塊 × AI 主題輪動 · 三大法人資金流」）；四象限統計由左側欄移至圖表上方、緊湊化並動態對齊圖表垂直零線；K 線區可用分隔線拖曳調整寬度（最多兩倍）；日期標籤釐清為「最新資料」（資料更新日）與「檢視日」（時間軸所在日），消除混淆；「逆勢強勢」鈕加「開/關」狀態標示、ⓘ 圖表說明與完整新手指南依現有功能重寫；頁尾顯示版本資訊與更新日期。
- **Y 軸動能改用 EMA 輕平滑，減少假象限翻轉**（R7）：對各板塊日淨買序列先做指數加權平滑（EMA，span=3）再照 5 日窗計算 X 與 Y。日與日之間的象限翻轉率由 **30.6% 降到 24.5%**（約少 1/5 假訊號）。因 X、Y 用同一條平滑序列同窗長，**`Y ≡ X(今) − X(五日前)` 恆等**（已驗證誤差 0），[ADR-0004](docs/decisions/ADR-0004-x-axis-daily-average.md) 的「Y＝X 的變化」自洽完全保留。圈大小 `sz`（近20日累計實際金額）**維持原始未平滑**。選 span=3（輕）而非 span=5（重）是為守住「早點看到苗頭」定位、避免拖慢真實轉折。設計理由見 [ADR-0005](docs/decisions/ADR-0005-ema-smoothing-momentum.md)。

### Docs
- ⓘ 圖表說明、新手指南、資訊卡、排行榜欄位標示 X 為「平滑趨勢值」並解釋（先 EMA 平滑再取均、壓單日大單雜訊），守住「不誤導」原則。
- 新增 [ADR-0005](docs/decisions/ADR-0005-ema-smoothing-momentum.md)；ROADMAP **R7** 標為已處理。

## [0.2.1] - 2026-06-18

X/Y 同單位的指標釐清（X 軸改日均）、縱軸說明誤導修正、以及 CI 維護（workflow action 版本升級）。泡泡座標位置與四象限歸屬皆不變，無破壞性變更。

### Changed
- **X 軸改為「近5日日均」（億/天）**，與 Y 軸同單位（R12）：`roll5()` 由加總改取平均，使「X＝位置、Y＝動能（X 的變化）」的 RRG 語意自洽。同步把對數刻度參數 `ltx` 由 5 改 1，**泡泡座標位置完全不變**（`x/5` 配 `ltx=1` 等價於原 `x` 配 `ltx=5`），僅顯示數字、刻度、軸標、資訊卡與排行榜欄位換成日均量級。圈大小 `sz`（近20日累計）維持總和（億）不變。設計理由見 [ADR-0004](docs/decisions/ADR-0004-x-axis-daily-average.md)。
- **CI：升級 GitHub Actions 至 Node 24 版本**（R11）：`actions/checkout@v4 → v5`、`actions/setup-python@v5 → v6`。GitHub 已宣布 2026-06-02 起 runner 預設強制 Node 24、2026-09-16 移除 Node 20，升級主版本以消除 deprecation 警告並維持自動更新管線運作。僅動 workflow，不影響資料計算與前端。

### Fixed
- 🟡 **縱軸說明誤導**：原頂部 tooltip 與長說明寫「上＝買盤加速、下＝放緩」，此絕對說法只在右半邊（在買）成立；左半邊（在賣）相反——往上＝賣壓趨緩、往下＝賣壓加速，與下方正確的四象限文字自相矛盾。改為**方向中性**說法：往上＝比上週更偏買（買更多或賣更少）、往下＝更偏賣，並把公式（本週日均淨買 − 上週日均·億/天）寫進說明，「加速／放緩」改註明須配合橫軸看。僅文字修正，未動 `accel()` 數學與四象限歸屬。

### Docs
- 在 ⓘ 圖表說明與新手指南加入「**X＝位置、Y＝動能**」心智模型、「為何用日均（與 Y 同單位）」、以及「**絕對 vs 相對：要公平比輪動該用相對強度**」的白話說明，依介面層級分配（glance 列最短、tooltip 中等、新手指南完整）。
- 新增 [ADR-0004](docs/decisions/ADR-0004-x-axis-daily-average.md)；ROADMAP **R12** 標為已處理、**R7** 註記「縱軸文字已先行釐清」。

## [0.2.0] - 2026-06-16

落實 [圖表系統審查報告](docs/archive/圖表系統審查報告.md) 的全部發現，並新增相對強度座標尺度。

### Added
- **座標尺度切換：金額 ↔ 相對強度**（F-V14）：右上新增 `normSel` 切換。相對強度模式以各板塊**自身日淨買超標準差 σ** 標準化（不減均值、僅除以 σ，故象限歸屬與絕對模式一致），消除「大型股永遠最大」的偏誤，讓「輪動」可跨板塊比較。資訊卡加顯「相對強度 σ」。設計理由見 [ADR-0003](docs/decisions/ADR-0003-relative-strength-normalization.md)。（原 ROADMAP R3 — 審查報告列為「最該補的一刀」）

### Fixed
- 🔴 **時間基準不同步**：修正時間軸停在兩交易日之間時，泡泡位置（內插 `S.pos`）與資訊卡（整數交易日 `S.cur`）講不同一天，導致象限/顏色與文字打架。採「非播放時 `setPos` 以 `Math.round` 吸附整數交易日」（F-V3, F-V5；見 [ADR-0002](docs/decisions/ADR-0002-time-base-alignment.md)）。

### Changed
- 資訊卡象限文字加上方位，例：「退潮（左下）」，由 `QORI` 提供，抗誤讀（F-V5；原 R2）。
- 縱軸正名：「資金加速度」→「資金動能（力道變化·億/天）」，避免誤解為物理二階導數（F-V1；原 R4）。
- 圖例註明雙週期：泡泡顏色＝近5日方向、大小＝近20日量級（F-V2；原 R5）。

### Docs
- 建立完整專案治理體系：產品定位、PRD、變更治理、流程與範本、ROADMAP、決策紀錄（ADR-0001/0002/0003）。
- 將《圖表系統審查報告》歸檔至 `docs/archive/`（發現已全數處理）；專案根目錄舊版 HTML 移入 `archive/`。

## [0.1.0] - 2026-06-16

首個有文件紀錄的基準版本（審查報告修正前的狀態）。

### Added
- **RRG 四象限輪動泡泡圖**（F-V1）：X＝近5日淨買超水位、Y＝資金動能、雙 log 軸，四象限（漲潮/輪動/觀望/退潮）。
- **泡泡編碼**（F-V2）：顏色反映近5日方向，大小反映近20日累計。
- **時間軸播放／拖曳 + 輪動軌跡**（F-V3）。
- **四象限統計卡，可點選篩選**（F-V4）。
- **泡泡資訊卡**（F-V5）：近5日淨買超、動能、象限。
- **龍頭股 K 線面板**（F-V6）：個股 OHLC／指數走勢線雙模式。
- **時間範圍選擇、板塊搜尋/篩選、顯示設定、資金熱力圖、說明彈窗、當日摘要列**（F-V7~V13）。
- **資料管線**（F-D1~D3）：`build_data.py` 從證交所 T86 + MI_INDEX 計算各產業資金流、類股指數、龍頭股 OHLC，純標準庫、零相依。
- **每交易日自動更新**（F-D4）：GitHub Actions 台灣時間約 18:30 自動跑、有變動才 commit。
- **離線備援**（F-V12）：`index.html` 內嵌資料，雙擊即可離線使用。

> 此版本已知的介面矛盾與方法論限制，已於 v0.2.0 全數處理。
