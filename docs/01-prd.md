# PRD — 產品功能規格

> **這份文件的角色**：把 [產品定位](00-product-brief.md) 的方向，拆成具體的功能清單，每個功能說明「對應什麼使用情境、目的是什麼、怎麼算做完」。要加或改功能前先更新這裡。

- 版本：對應產品 v0.3.0
- 最後更新：2026-06-18
- 維護者：單人

---

## 1. 範圍與架構

產品由兩個層次組成，**刻意分離**（見 [定位 §5 設計原則 4](00-product-brief.md)）：

- **資料層**：`build_data.py` 從證交所抓 T86（三大法人買賣超）+ MI_INDEX（每日收盤行情），算出各產業資金流、類股指數、龍頭股 OHLC，輸出 `data.json`。由 GitHub Actions 每交易日自動跑。
- **呈現層**：`index.html`（單檔前端）讀 `data.json` 渲染 RRG 輪動圖與 K 線。內建一份備援資料，可離線開啟。

`data.json` 的 schema 是兩層之間的契約（見 §4）。

## 3. 功能清單

格式：`ID｜名稱` → **目的** ＋ **驗收標準**（可勾選的「做完」定義）＋ 狀態。

狀態圖例：✅ 已上線　🟡 已上線但有已知問題　⬜ 規劃中（見 [ROADMAP](ROADMAP.md)）

### 3.1 資料層

**F-D1｜分產業三大法人淨流計算** ✅
目的：UC1/UC2 的資料基礎。為每個上市產業算出每日三大法人淨買超（億元）。
驗收：對證交所 38 個產業代碼逐一抓 T86；以「股票成交金額×收盤價近似」彙總到產業；無資料產業自動跳過；輸出單位為億元、四捨五入 2 位。

**F-D2｜板塊龍頭股 OHLC** ✅
目的：UC5 量價對照。為每個產業挑「區間成交金額最大」的龍頭股，附每日 OHLC。
驗收：龍頭以區間累計成交金額取最大；OHLC 缺值（停牌/無交易）以 `null` 表示，前端要能處理。

**F-D3｜類股指數收盤序列** ✅
目的：指數模式下的板塊走勢參照。
驗收：從 MI_INDEX 解析「XX類指數」收盤；產業名與指數名做模糊對應；無對應指數時為 `null`。

**F-D4｜每交易日自動更新** ✅
目的：UC7 資料永遠新，零人工。
驗收：GitHub Actions 於每週一～五 UTC 10:30（台灣 18:30）執行 `build_data.py`；`data.json` 有變動才 commit；具 `workflow_dispatch` 可手動觸發。

**F-D5｜穩定的 data.json 契約** ✅
目的：資料層/呈現層解耦。
驗收：schema 變動視為破壞性變更（需升 MINOR/MAJOR 並記 CHANGELOG）；schema 定義見 §4。

### 3.2 呈現層

**F-V1｜RRG 四象限輪動泡泡圖** ✅
目的：UC1/UC2/UC4 核心。以 X=近5日水位、Y=資金動能、雙 log 軸繪製各板塊泡泡。
驗收：四象限切法 `quad(x,y)` 與圖面四角標籤一致（右上漲潮/右下輪動/左上觀望/左下退潮）；零軸清楚；log 刻度。支援兩種座標尺度（金額／相對強度，見 F-V14）。退潮時的「逆勢強勢」金色高亮需相對強度 ≥ 1σ 才入選（與中性門檻一致，見 [ADR-0009](decisions/ADR-0009-counter-tide-sigma-floor.md)）。

**F-V2｜泡泡顏色與大小編碼** ✅
目的：一個泡泡同時傳達方向（5日）與量級（20日）。
驗收：顏色 `col(x)` 反映近5日方向（流入橘/流出綠/中性灰）；**中性灰的判定改用各板塊自身波動 σ（`|近5日日均| < 1σ` 即中性），取代原本的固定金額門檻，兩座標模式共用同一規則——見 [ADR-0008](decisions/ADR-0008-neutral-threshold-sigma.md)**；大小反映近20日累計；圖例已明示「顏色＝近5日方向、大小＝近20日量級」的雙週期含義（原 R5 已完成）。

**F-V3｜時間軸播放／拖曳 + 輪動軌跡** ✅
目的：UC3/UC4，看板塊在象限間的順時針移動。
驗收：可播放動畫、可拖曳時間軸；顯示近期 trail 路徑；**非播放狀態下泡泡吸附整數交易日**（`setPos` 在 `!fromPlay` 時 `Math.round`），與資訊卡用同一時間基準，不再出現泡泡與文字打架（原 P0 bug 已修，見 [ADR-0002](decisions/ADR-0002-time-base-alignment.md)）。

**F-V4｜四象限統計卡（可篩選）** ✅
目的：UC1 快速看各象限有幾個板塊；點卡片篩選圖上泡泡。
驗收：四張卡（漲潮/輪動/觀望/退潮）各顯示數量；點選可高亮/篩選對應象限板塊。

**F-V5｜泡泡資訊卡** ✅
目的：UC3/UC5 看單一板塊細節。
驗收：顯示近5日淨買超、近20日累計、動能（力道變化）、所屬象限**含方位**（如「退潮（左下）」，由 `QORI` 提供，原 R2 已完成）；相對強度模式下另顯示「相對強度 σ」。

**F-V6｜龍頭股 K 線面板（個股/指數雙模式）** ✅
目的：UC2/UC5 量價對照。
驗收：個股模式顯示龍頭股真實 OHLC K 線；指數模式顯示類股指數走勢線（指數無 OHLC）；可切換（ktoggle）；無資料時顯示空狀態提示。

**F-V7｜時間範圍選擇** ✅
目的：切換觀察窗（如 20/40/60 日）。
驗收：切換後圖與 K 線同步重繪；目前選項在 UI 高亮。

**F-V8｜板塊搜尋／篩選** ✅
目的：UC2 快速定位特定板塊。
驗收：可輸入關鍵字篩選；可全選/全不選/清除。

**F-V9｜顯示設定（字級等）** ✅
目的：可讀性調整。
驗收：字級調整即時生效。

**F-V10｜資金熱力圖** ✅
目的：補充視角，以熱力色塊看板塊資金強弱。
驗收：與主圖資料一致。

**F-V11｜「怎麼看這張圖」說明** ✅
目的：UC6，降低誤讀（呼應 [定位 §5 原則 2 誠實的不確定性](00-product-brief.md)）。
驗收：說明四象限、軸義、「減速 ≠ 倒車」等；含白話例子。

**F-V12｜手動重新整理 + 離線備援** ✅
目的：UC7。部署後優先讀同目錄最新 `data.json`，右上 🔄 可手動重抓；離線時用內嵌備援資料。
驗收：`file://` 直開可用備援；HTTP 部署優先讀外部 data.json。

**F-V13｜當日資金摘要列** ✅
目的：UC1 一行看懂大方向。
驗收：顯示當日整體淨買/賣超等摘要。

**F-V14｜座標尺度切換：金額 ↔ 相對強度** ✅
目的：UC2/UC4 核心強化。消除「大型股永遠最大、小板塊擠在中間」的偏誤，讓「誰在輪動」真正可比（落實 [定位 §4.4 限制 1](00-product-brief.md#44-已知的方法論限制誠實揭露) 的改善）。
驗收：右上 `normSel` 提供「金額（億）／相對強度（×自身波動）」切換；相對強度模式以各板塊**自身日淨買超標準差 σ** 為尺度做標準化（`mnorm`／`sigma`）；**不減均值、僅除以 σ**，故正負號不變、象限歸屬與絕對模式一致；切換時座標軸刻度與標籤同步換成 `rel` 設定；資訊卡加顯「相對強度 σ」。設計理由見 [ADR-0003](decisions/ADR-0003-relative-strength-normalization.md)。兩種模式共用同一個 **1σ 中性門檻**（見 [ADR-0008](decisions/ADR-0008-neutral-threshold-sigma.md)），確保同一板塊「是否中性」在兩模式下一致。

## 4. 資料契約（data.json schema）

資料層與呈現層之間的介面。**任何 schema 變更都是破壞性變更**，需升版並記 CHANGELOG。

```jsonc
{
  "updated": "YYYYMMDD",                 // 最新交易日
  "dates":   ["YYYYMMDD", ...],          // 由舊到新
  "sectors": [
    {
      "sector": "半導體",
      "series": [float, ...],            // 各日三大法人淨買超(億元)，對齊 dates
      "idx":    [float|null, ...],       // 類股指數收盤，對齊 dates
      "rep":    {"id":"2330","name":"台積電"},  // 區間成交金額最大龍頭股，可為 null
      "ohlc":   [[o,h,l,c]|null, ...]    // 龍頭股每日 OHLC，對齊 dates
    }
  ]
}
```

關鍵常數（前端計算指標用）：`WIN=5`（水位視窗）、`BASE=5`（動能基期）、`CUM=20`（泡泡大小累計）。改這些常數會改變圖的意義，視為功能變更。
