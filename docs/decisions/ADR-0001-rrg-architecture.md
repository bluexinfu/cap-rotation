# ADR-0001：以 RRG「水位 × 動能」四象限呈現板塊資金流

- 狀態：已採納
- 日期：2026-06-16（追溯記錄產品既有的核心設計）
- 相關：F-V1、F-D1、[產品定位 §4 理論基礎](../00-product-brief.md#4-理論基礎這張圖在做什麼)

## 背景
散戶看三大法人資料只能看到靜態排行榜，看不到「資金正在流入/流出哪些板塊、力道在加速還是放緩」這種動態與輪動資訊。需要一個能呈現「資金生命週期」的視覺化方式。

## 決定
採用 RRG（Relative Rotation Graph）式的二維輪動圖作為核心視圖：
- X 軸 = 近 5 日三大法人淨買超水位（`WIN=5`）
- Y 軸 = 資金動能 = 近 5 日日均 − 前 5 日日均（`BASE=5`）
- 泡泡大小 = 近 20 日累計（`CUM=20`），顏色 = 近 5 日方向
- 四象限：漲潮/輪動/觀望/退潮，板塊在象限間順時針循環

## 理由
- 「水位 × 動能」拆兩軸，能提早看到「仍淨買但力道轉弱」的早期退場訊號，這是排行榜給不出的。
- 用動能而非單日金額，能濾雜訊、看趨勢。
- 順時針軌跡符合資金輪動的生命週期，配合時間軸播放可看出板塊在循環裡的位置。

## 後果
- 好處：把「資金流向與節奏」變成看得見的軌跡，支撐 UC1/UC2/UC3/UC4。
- 代價 / 風險：
  - 用**絕對金額、未標準化** → 大型股偏誤（半導體永遠最大），削弱「輪動可比性」。列為 [ROADMAP R3](../ROADMAP.md) 的優先改善項。
  - 5 日視窗短 → 動能敏感，一筆大單可能翻象限。
  - 雙 log 軸使「離零距離」視覺非線性，解讀強弱要留意。
- 連帶：定義了 `data.json` 的 `series` 欄位與前端三個計算常數。

## 考慮過的替代方案
- **純排行榜/熱力圖**：實作簡單，但看不到動態與輪動，無法支撐 UC3/UC4。（熱力圖最後作為輔助視圖保留，見 F-V10。）
- **單軸時間序列折線**：能看趨勢但無法同時表達「水位 + 動能」兩個維度。
