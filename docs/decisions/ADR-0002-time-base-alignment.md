# ADR-0002：非播放時泡泡吸附整數交易日，消除時間基準不同步

- 狀態：已採納（已於 v0.2.0 上線：`setPos` 在 `!fromPlay` 時 `Math.round`）
- 日期：2026-06-16
- 相關：F-V3、F-V5、[PRD §5](../01-prd.md#5-v020-已完成審查報告修正清單)、[圖表系統審查報告 §1](../archive/圖表系統審查報告.md)

## 背景
時間軸停在兩個交易日之間時，會出現「泡泡畫在右下、橘色（流入），但資訊卡寫近5日 −175 億、象限退潮」的矛盾。根因是兩套時間基準不同步：
- 泡泡位置 `updateMapAt(S.pos)` → `metricsAt(sec, S.pos)`，用**連續內插的小數時間** `S.pos`（如 37.4）。
- 資訊卡 `showTip()` → `metrics(sec, S.cur)`，用 `S.cur = Math.round(S.pos)`，**四捨五入到最近的真實交易日**。

當某板塊隔日有一筆大額買超，內插後的泡泡 x 被拉成正值（右側、橘色），但資訊卡回到當日顯示負值（退潮）。畫面與文字講的是不同一天。

## 決定
**非播放狀態下，`updateMapAt` 改用 `Math.round(S.pos)`**，讓泡泡與資訊卡永遠對齊在同一個真實交易日。內插只在播放動畫進行中才使用。

## 理由
- 對投資判斷而言，停下來看的應該是「真實某一天」的狀態，而不是不存在的內插中間值。
- 從根源讓「位置、顏色、數字」三者用同一個時間基準，徹底消除矛盾，而非只修表面。
- 比替代方案（讓資訊卡也跟著內插）更符合「呈現真實資料、不營造虛假精確感」的[設計原則](../00-product-brief.md#5-設計原則取捨時的優先序)。

## 後果
- 好處：解掉使用者看到的矛盾；強化資料可信度。
- 代價 / 風險：拖曳時泡泡會「吸附」到最近交易日、不再連續滑動——可接受，因為小數日的數字本來就不是真實資料。
- 連帶：CHANGELOG 記 Fixed；PRD F-V3/F-V5 的已知問題標記可移除。

## 考慮過的替代方案
- **方案 B：`showTip` 改用內插值** `metricsAt(sec, S.pos)`，並用同一組 x、y 算象限，讓卡片跟著泡泡走。也能消除矛盾，但會讓資訊卡顯示「不存在的內插中間值」，與設計原則相悖。故不採。
- **兩者都做**：審查報告建議可同時做，但以方案 A（本決定）為主即足以解掉矛盾，先求簡單。
