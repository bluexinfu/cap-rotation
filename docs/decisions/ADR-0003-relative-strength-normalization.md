# ADR-0003：以「各板塊自身波動 σ」做相對強度標準化（不減均值）

- 狀態：已採納（已於 v0.2.0 上線）
- 日期：2026-06-16
- 相關：F-V14、F-V1、F-V5、[PRD §5](../01-prd.md#5-v020-已完成審查報告修正清單)、[圖表系統審查報告 §4.2](../archive/圖表系統審查報告.md)

## 背景
原本 X 軸與泡泡大小用**絕對金額（億）**，造成大型股偏誤：半導體因台積電金額天生最大，永遠佔右側、泡泡最大，小型板塊再怎麼轉強也擠在中間。但「輪動」本質是**相對**的——我們想看「誰相對自己平常更受青睞」，而不是「誰絕對金額大」。這是審查報告點名「最該補的一刀」。

審查報告提了兩個方向：(a) 淨買超 ÷ 該類股區間成交金額；(b) 對各板塊自身歷史做 z-score。

## 決定
新增「金額 ↔ 相對強度」座標尺度切換（`normSel`）。相對強度模式以**各板塊自身日淨買超的標準差 σ**（`sigma(sec)`）為尺度，把指標除以 σ 後再繪圖（`mnorm`）：
- **只除以 σ、不減均值**（不是完整 z-score）。
- 預設仍是「金額」模式；相對強度為可選視角。
- 切換時座標軸刻度與標籤同步換成 `rel` 設定；資訊卡加顯「相對強度 σ」。

## 理由
- 用「自身波動」當尺，等於問「這個流入相對它平常的波動算大嗎」，直接消除大型股偏誤、讓板塊間可比。
- **刻意不減均值**：保留正負號，使「右＝淨買、左＝淨賣」與象限歸屬在兩種模式下**一致**，不會出現「金額模式在右、相對模式跳到左」的認知衝突。這是相對於完整 z-score 的關鍵取捨。
- 選 σ 而非「÷成交金額」：σ 只需板塊自身的淨買序列即可算，不需額外引入成交金額分母，維持資料管線簡單；且更貼近「相對自身常態」的語意。
- 以「可切換的選項」而非「取代金額」呈現：金額仍是「真實資金量」的有用資訊，符合[設計原則 2 誠實揭露](../00-product-brief.md#5-設計原則取捨時的優先序)，把選擇權交給使用者。

## 後果
- 好處：讓「誰在輪動」真正可跨板塊比較，補上產品最大的方法論缺口（[定位 §4.4 限制 1](../00-product-brief.md#44-已知的方法論限制誠實揭露) 由此降級為「已處理」）。
- 代價 / 風險：
  - 多一個模式 = 多一份認知負擔，需靠 `normSel` 的 tooltip 與說明彈窗解釋清楚。
  - σ 對歷史視窗長度敏感；資料天數少時 σ 估計不穩。
  - 「不減均值」使它不是教科書 z-score，未來若有人想要真 z-score，需另開討論。
- 連帶：PRD 新增 F-V14；新增座標設定 `rel`；資訊卡欄位增加。

## 考慮過的替代方案
- **完整 z-score（減均值再除以 σ）**：統計上更標準，但會改變正負號與象限歸屬，與金額模式不一致、易誤讀。不採。
- **÷ 區間成交金額**：也能消除偏誤，但需引入成交金額分母、增加資料管線複雜度，且語意是「滲透率」而非「相對自身常態」。列為未來可比較的替代，暫不採。
- **直接用相對強度取代金額（不保留切換）**：失去「真實資金量」資訊，違反誠實揭露原則。不採。
