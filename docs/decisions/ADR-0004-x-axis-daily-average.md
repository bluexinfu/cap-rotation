# ADR-0004：X 軸由「近5日總和」改為「近5日日均」（與 Y 軸同單位）

- 狀態：已採納（[Unreleased]，待發版）
- 日期：2026-06-17
- 相關：R12、R7、F-V1、[ADR-0001 RRG 架構](ADR-0001-rrg-architecture.md)、[圖表系統審查報告](../archive/圖表系統審查報告.md)

## 背景

這張圖是 RRG（相對輪動圖）的變體，其核心邏輯是：**X 軸＝位置（資金現在在哪），Y 軸＝動能（這個位置正在怎麼變，即 X 的變化率）**。要讓「順時針輪動」與四象限邏輯成立，Y 最好是 **X 這同一個量的時間變化**。

但原實作不一致：

| | 原本 | 問題 |
|---|---|---|
| X | 近5日淨買超**總和**（億） | 是「總量」 |
| Y | 本週日均 − 上週日均（億/天） | 是「速率之差」 |

X 是總量、Y 是速率之差，單位不一致，Y 嚴格說不是 X 的導數（差了一個固定倍數 5）。輪動形狀仍大致正確（總和與日均同向、同號），但這是框架上的小瑕疵，也讓 tooltip 必須用「日均」舉例、與 X 的「總和」定義對不上，增加理解負擔。

釐清：5 日**總和 = 5 ×日均**，兩者只差固定倍數，**平滑度與正負號完全相同**——平滑來自「取 5 日窗」本身，與「總和或日均」無關。故總和 vs 日均純粹是**單位選擇**，不影響任何訊號。

## 決定

X 軸改為 **近5日「日均」淨買超（億/天）**：`roll5()` 由加總改為取平均（`/WIN`）。

同步調整：
- 對數刻度參數 `ltx` 由 `5` 改為 `1`。因 `x_新 = x_舊 / 5`、`txf=sign·log10(1+|x|/ltx)`，故 `txf(x_舊/5, ltx=1) ≡ txf(x_舊, ltx=5)`——**泡泡座標位置完全不變**，僅顯示數字與刻度換成日均量級。
- `TKCFG.abs.xt` 刻度改為日均量級；`xlab` 改「億/天」。
- 資訊卡「近5日淨買超 … 億」改「近5日日均 … 億/天」；排行榜欄位「近5日(億)」改「近5日日均」。
- 圈大小 `sz`（近20日累計）**維持總和（億）**——它是「中期進出的量級」，本就是累積量、非速率，不改。

## 理由

- X、Y 同單位（億/天），Y 真正成為 X 的變化，RRG「位置＋動能」語意自洽。
- 與 tooltip／新手指南既有的「日均」舉例對齊，理解負擔下降。
- 因座標位置不變（見上），屬**零視覺風險**的純定義／單位修正，不影響既有判讀與象限歸屬。

## 後果

- 好處：指標定義一致、說明更好懂；為 R7（動能平滑、減少假訊號）打底——未來在一致的單位上談平滑更清楚。
- 代價／風險：
  - 屬指標定義變更，慣看舊「總和」數字的使用者，數字會變成 1/5（同一天 +100 億總和 → +20 億/天）；已在說明標注單位「億/天」降低誤讀。
  - 與圈大小 `sz` 仍用「總和（億）」並存，需注意資訊卡單位不同（日均 vs 累計），已分別標單位。

## 考慮過的替代方案

- **維持 X＝總和、只修文字**：成本最低，但框架不一致的根因仍在，tooltip 與定義長期對不上。不採。
- **X、Y 都改標準化（z-score／÷σ）**：統計上更嚴謹、跨板塊更可比，但會犧牲「實際幾億」的直覺。已以 [ADR-0003](ADR-0003-relative-strength-normalization.md) 的「相對強度」可切換模式涵蓋此需求；絕對模式維持直覺定位即可。不在本次取代。
