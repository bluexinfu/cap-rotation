# ADR-0005：對資金序列做 EMA(span=3) 輕平滑，減少假象限翻轉

- 狀態：已採納（[Unreleased]，待發版）
- 日期：2026-06-18
- 相關：R7、F-V1、[ADR-0001 RRG 架構](ADR-0001-rrg-architecture.md)、[ADR-0004 X 軸日均](ADR-0004-x-axis-daily-average.md)

## 背景

四象限的歸屬由 X（位置）與 Y（動能）的正負決定。原指標以 5 日窗計算，視窗短、對單日大單敏感：**一筆異常的單日法人買賣超就能把板塊翻到另一個象限**。用現有 data.json 量化，**日與日之間板塊象限的翻轉率高達 30.6%**（近三天一次），對一個訴求「看資金輪動趨勢、非盤中訊號」的工具是明顯的雜訊（假訊號）。

R7 的目標就是降低這類假訊號，但不能破壞 [ADR-0004](ADR-0004-x-axis-daily-average.md) 剛建立的「**Y 恰為 X 的週對週變化**（X、Y 同單位、同 5 日窗）」自洽，也不能違反產品「不誤導、可解釋」的設計原則。

## 決定

對每個板塊的**日淨買序列先做指數加權平滑（EMA，span=3）**，再照原本的 5 日窗計算 X 與 Y：

- `X = roll5(EMA(series))`、`Y = accel(EMA(series))`（視窗仍 5/5）。
- 因 X、Y 用同一條平滑序列、同窗長，**`Y ≡ X(今) − X(五日前)` 恆等**（已驗證最大誤差 0.00），ADR-0004 的自洽完全保留。
- 圈大小 `sz`（近20日累計）**維持原始未平滑**——它代表「實際進出多少錢」，平滑會使金額失真。
- 相對強度模式的 σ（[ADR-0003](ADR-0003-relative-strength-normalization.md)）暫維持以**原始日淨買**計算（σ 的基準屬 R8 範疇，本次不動）。

實作：新增 `EMA_SPAN=3` 與 `emaSeries(sec)`（含快取 `sec._sm`）；`metrics()` 的 x/y 改讀平滑序列、sz 讀原始序列。

## 理由

- **有效且不過頭**：span=3 把翻轉率由 30.6% 降到 **24.5%**（約少 1/5 假訊號）。
- **守住「早點看到苗頭」定位**：驗證發現較重的 span=5（翻轉率 20.6%）雖砍最多雜訊，但會**拖慢真實轉折**——以半導體為例，最近 3 日資金明顯灌回（+263/+273/+438 億），span=5 仍顯示「退潮」，等於用「反應太慢的假陰性」換掉「太跳的假陽性」，與本工具「早點看到苗頭」的定位矛盾。span=3 則顯示「觀望（流出趨緩、回穩中）」，既承認轉折又不過度宣告，最貼合定位。
- **誠實標示**：平滑後 X 不再是純算術 5 日均，故資訊卡、排行榜欄位標「(平滑)」，ⓘ 說明與新手指南明述「先做 EMA 平滑再取均、是平滑後的趨勢值」，符合「不誤導」原則。

## 後果

- 好處：象限與泡泡位置更穩、輪動軌跡更平順，減少被單日大單誤導。
- 代價／風險：
  - 仍是**指標定義變更**：發版時最新交易日約 8/32 板塊象限會與舊版不同（多為臨界、近軸的板塊），慣看舊圖者會察覺差異，已以說明標注。
  - EMA 仍有輕微落後（本質如此），在極端急轉的板塊會比純 5 日均晚一步反映；這是降噪的必要代價，已以最輕的 span=3 把落後壓到最小。

## 考慮過的替代方案

- **對稱拉長窗口（10/10）**：翻轉率降到 21.0% 且守住 Y=ΔX，但反應慢約一週、且須把全 UI「近5日」改「近10日」，與「早點看到苗頭」定位不符。不採。
- **不對稱拉長 BASE（5/20）**：數據顯示砍假訊號最差（僅到 26.2%）**又**破壞「Y＝X 的變化」自洽，兩頭皆輸。淘汰。
- **EMA 點對點（X=EMA、Y=EMA今−EMA五日前）**：看似乾淨單步，但點對點差分本身更抖，span=5 翻轉率反而升到 37.2%（比現況還糟）。淘汰。
- **EMA span=5（重平滑）**：砍雜訊最多（20.6%）但拖慢轉折、與定位矛盾（見理由）。不採。
- **中性門檻 σ 死區（R8）**：另一條減少臨界翻轉的路，與本案互補；屬指標定義變更、且 σ 基準會受未來「新分類維度」影響，刻意延後、不在本案處理。
