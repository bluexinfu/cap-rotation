# ADR-0007：大盤潮汐 + 逆勢強勢高亮（相對強度排名）

- 狀態：已採納（[Unreleased]，待發版）
- 日期：2026-06-18
- 相關：F-V1、R9（量價背離）、[ADR-0003 相對強度](ADR-0003-relative-strength-normalization.md)、[ADR-0006 AI 主題維度](ADR-0006-ai-theme-dimension.md)、`market` 資料（[build_data.py](../../build_data.py)）

## 背景

使用者的洞察：全市場三大法人買賣超像整片海的潮水（漲潮＝法人買大盤、退潮＝法人賣大盤），各板塊/主題都在這片海裡。多數板塊會**跟著潮水走**；真正有意義的不是「它在被買」，而是「**在大盤退潮時，它逆勢還在被買、而且力道還在擴大**」——這通常是法人不顧大盤、堅定押注的強勢/抗跌標的。

這是一種「**相對市場的強度**」，與工具既有的「相對強度(÷σ)」模式（拿板塊跟它自己的波動比）不同——本案是拿板塊**跟大盤潮水**比。

## 決定

1. **大盤潮汐狀態**：摘要列顯示「大盤潮汐 漲潮/退潮 ±X 億/天」（全市場三大法人近5日日均買賣超；資料來自 `data.json` 的 `market.flow`）。
2. **逆勢強勢高亮（可切換，預設開）**：頂部「🌊 逆勢強勢」鈕。當**大盤退潮（潮汐<0）**時，把符合「逆勢」條件的板塊/主題用金色高亮 + 🌊 標示並強制顯示標籤。
3. **逆勢條件 + 排名**：候選＝自身近5日日均淨買 `x>0`（仍淨流入）且動能 `y>=0`（力道沒減速）；再從候選中**只取相對強度 `x/自身σ` 最強的前 5 名**（`COUNTER_TOPN=5`）。摘要列直接列出這幾個名字。
4. 兩個維度（證交所產業、AI 主題）都適用；屬**觀察用高亮，非買賣訊號**。

## 理由

- **為何要排名、而非「>0 就標」**：用真實資料驗證，naive 的「自身正、大盤負」規則在大型股殺盤日會標出多達 **19/32** 個板塊——因為大盤總額被大型股（台積電/半導體）主導，當它們殺盤把大盤拖成退潮，許多「只是沒參與那波賣壓」的中小板塊就被誤判為逆勢強勢。這不是「強」，只是「沒被賣」。
- **為何用 `x/σ`（相對強度）排名**：消除大小偏誤（大型股淨買天生大），讓「最強」是**相對自身波動**最突出者，公平且自動限制數量。實測收斂後，例如 2026-06-12 退潮，最強為數位雲端 2.4σ，明顯領先，訊號利。
- **為何要 `y>=0`**：呼應使用者「退潮時**依然在擴大**」，排除「還在買但力道正在縮」的板塊。

## 後果

- 好處：把「相對市場的強度」這個強訊號視覺化，退潮時一眼看到誰最抗跌/最被堅定買進；補上既有「相對強度(÷σ)」模式所沒有的「對照大盤」維度。
- 限制／須知：
  - **只有退潮時才會高亮**；漲潮日（如最新交易日常見）不顯示，屬正常。
  - 「最強前5」是**相對排名**，弱勢盤裡的第一名 σ 值也可能不高（之後可加「最低 σ 門檻」精煉，本次未做）。
  - 它是**觀察輔助、非選股建議**（守住非目標）；只標群組/板塊層級的相對位置，不對個股給評分。

## 考慮過的替代方案

- **naive「自身正、大盤負」全標**：訊號太鬆（最多 19/32），淘汰，改用相對強度排名。
- **絕對金額門檻（x 大於某億數才標）**：會偏袒大型股，違反公平，不採；改用 ÷σ。
- **限定漲潮象限（x>0 且 y>0 強加速）**：可考慮的更嚴格版本；本次先用 `y>=0`（不減速）保留多一點候選，再靠排名收斂。
- **加最低 σ 門檻**：避免弱勢盤標出不夠強的「最強」；列為日後精煉，未在本次。
