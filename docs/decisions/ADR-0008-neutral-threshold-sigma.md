# ADR-0008：四象限中性門檻改用各板塊自身波動 σ（取代固定金額）

- 狀態：已採納（[Unreleased]，待發版）
- 日期：2026-06-27
- 相關：F-V1、F-V2、F-V14（PRD）、R8（[ROADMAP](../ROADMAP.md)）、[ADR-0003 相對強度標準化](ADR-0003-relative-strength-normalization.md)

## 背景

四象限圖用「中性灰」表示一個板塊近期資金沒有明顯方向。原本中性判定用**固定門檻**：
- 金額模式：`|近5日日均| < 3 億` → 中性灰。
- 相對強度模式：`|x/σ| < 0.3` → 中性灰。

問題在於各板塊「體型」差距極大——自身日淨買標準差 σ 從造紙約 0.5 億到半導體約 303 億，相差約 600 倍。同一個 ±3 億門檻套上去意義完全不同：

| 板塊 | 自身 σ(億) | 3億=幾σ | 中性%(固定) |
|---|---:|---:|---:|
| 半導體 | 303 | 0.01σ | 1% |
| 金融保險 | 37.9 | 0.08σ | 8% |
| 化學 | 8.9 | 0.34σ | 69% |
| 油電燃氣 | 3.1 | 0.97σ | 94% |
| 造紙 | 0.5 | 5.9σ | 100% |

大板塊在固定門檻下幾乎永遠「亮」（半導體僅 1% 時間中性），小板塊則永遠「灰」（造紙 100% 中性、從不顯示方向），且卡在門檻附近的板塊會反覆閃爍。真實資料（70 個交易日、32 個上市產業）：固定門檻下各板塊「中性%」從 1% 到 100%、離散度（標準差）=36；全體「分類翻轉率」=14.4%。

## 決定

中性門檻改用「**各板塊自身波動 σ**」為尺：**`|近5日日均| < 1×σ(該板塊)` 即視為中性**。兩種座標模式（金額／相對強度）**統一採同一個 1×σ 規則**，使同一板塊在兩模式下「是否中性」一致。σ 沿用既有 `sigma()`（板塊自身日淨買序列的標準差），不新增任何資料管線。

實作方向：前端 `col()` 改為依板塊 σ 判定中性——`thr = S.norm ? 1.0 : 1.0*σ(該板塊)`；相對強度模式的門檻由 0.3 提高到 1.0 以對齊；更新 `col()` 的兩個呼叫點（泡泡圖、摘要表）傳入板塊資訊。純前端變更，`build_data.py` 與 `data.json` schema 不動。

## 理由

- **公平**：用各板塊自己的波動當尺，問的是「這筆流入相對它自己平常算大嗎」，消除體型偏誤。實測「中性%」離散度由 36 收斂到 7（1×σ 下各板塊中性% 落在 68%~99%）。
- **更穩、少假動作**：全體「分類翻轉率」由 14.4% 降到 6.6%（約砍半）。
- **取 1.0 而非其他值**：0.5σ 反而升到 18%（門檻太窄、更敏感）；1.5σ 雖最穩（1.7%）但過度中性、會蓋掉真訊號。1.0σ 是「公平＋穩定」的平衡點。
- **語意乾淨**：顏色只在「資金流大過該板塊自身日常波動」時才亮＝真的值得注意才標。
- **零額外相依**：重用 [ADR-0003](ADR-0003-relative-strength-normalization.md) 已導入的 σ。

## 後果

- 好處：跨板塊公平可比、畫面更穩、訊號更誠實；兩座標模式的中性判定一致。
- 代價／須知：
  - 大型板塊（半導體、金融等）在平常日子會由「常亮」變「常灰」，只有爆量日才亮——屬**預期內的正確行為**，但與舊視覺習慣不同，需在使用者說明中講清楚。
  - σ 對歷史視窗長度敏感，天數少時門檻會抖（沿用 `sigma()` 既有限制）。
  - 相對強度模式變得比現在更「灰」（門檻 0.3→1.0），為換取兩模式一致的取捨。
- 連帶要改：PRD（F-V1/F-V2）、`index.html` 的 `col()` 與「怎麼看這張圖」使用者說明、CHANGELOG、ROADMAP（R8 標完成）。

## 考慮過的替代方案

- **維持固定金額（±3 億）**：對不同體型板塊不公平（中性% 1%~100%），淘汰。
- **0.5×σ**：更敏感但翻轉率反升到 18%，不採。
- **1.5×σ**：最穩（1.7%）但過度中性、可能蓋掉真訊號，不採。
- **金額模式與相對模式各用不同 k**：會造成同一板塊「是否中性」在兩模式不一致，違反 ADR-0003 追求的兩模式一致性，不採。
