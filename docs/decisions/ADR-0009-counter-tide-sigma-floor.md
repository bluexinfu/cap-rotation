# ADR-0009：逆勢強勢加最低 1σ 門檻（與 R8 中性門檻對齊）

- 狀態：已採納（[Unreleased]，待發版）
- 日期：2026-06-27
- 相關：F-V1（逆勢強勢）、R14（[ROADMAP](../ROADMAP.md)）、[ADR-0007 逆勢強勢](ADR-0007-counter-tide-strength.md)、[ADR-0008 中性門檻σ](ADR-0008-neutral-threshold-sigma.md)

## 背景

逆勢強勢（[ADR-0007](ADR-0007-counter-tide-strength.md)）：大盤退潮時，把「仍逆勢被買」的板塊中相對強度（x/自身σ）前 5 名用金色高亮。但選取**沒有最低門檻**——弱盤裡淨買板塊一少，連 +0.3σ、+0.5σ 的「矮子裡的高個」也會上金。

實測最近退潮日 2026-06-15（大盤潮汐 tide=−220）：無門檻前 5 名為 **數位雲端 2.56σ、建材營造 1.08σ、運動休閒 0.92σ、生技醫療 0.86σ、造紙 0.85σ**——5 個裡有 **3 個 < 1σ**。使用者在 2026-06-26（退潮）也看到造紙（+0.3σ）、資訊服務（+0.5σ）上金，質疑「這也算強？」。

更關鍵：[ADR-0008](ADR-0008-neutral-threshold-sigma.md)（R8）已把「< 1σ」定義為「相對自身波動還不算真的動」（填中性灰）。逆勢強勢卻把同樣 < 1σ 的板塊標成「強勢」——**兩功能對「什麼叫有動」的標準互相打架**。

## 決定

逆勢強勢候選加最低門檻：**相對強度（原始 x / 自身σ）≥ 1σ** 才能入選 top-5。門檻沿用 R8 的 1σ，使兩功能「什麼叫真的動」一致。同時把候選的相對強度計算改成**與座標模式無關**（`rel = S.norm ? m.x : m.x/σ`），順手修掉原本在相對強度模式下排序尺度不一致的小問題。

實作：`counterTops()` 新增常數 `COUNTER_MIN_SIGMA=1.0`；正確計算 `rel`；候選條件改為 `isCounter(m,tide) && rel >= COUNTER_MIN_SIGMA`，並以 `rel` 排序。

## 理由

- **一致**：與 R8 同一把「1σ＝真的動」的尺，消除「一個說沒動、一個說很強」的矛盾。
- **誠實**：弱盤裡若真的沒有 ≥ 1σ 的強勢，金色就不顯示——沒有就不硬標。實測 6/15 由 5 個收斂到 2 個真強勢（數位雲端、建材營造）。
- 順帶修正相對強度模式下的候選排序尺度。

## 後果

- 好處：金色只標真正強勢、跨座標模式一致、與 R8 對齊。
- 代價／須知：金色高亮會比以前**少出現**（弱盤日可能一個都沒有）——屬預期內的正確行為。門檻 1σ 是與 R8 對齊的選擇，未來若要更寬／嚴需另議。
- 連帶改：PRD（F-V1 逆勢強勢）、`index.html` ⑤ 使用者說明、CHANGELOG、ROADMAP（R14 標完成）。

## 考慮過的替代方案

- **維持無門檻**：把弱盤矮個誤標強勢（造紙 0.3σ），與 R8 矛盾，不採。
- **門檻取別的值（如 0.5σ）**：與 R8 的 1σ 不一致，仍會標到 R8 認定「沒動」的板塊，不採。
- **改用固定金額門檻**：回到大小板塊偏誤，違反 ADR-0003／0008 精神，不採。
