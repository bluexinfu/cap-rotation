# ADR-0010：新增「市場相對」第 3 種座標尺度（扣掉大盤共同潮水）

- 狀態：已採納（[Unreleased]，待發版）
- 日期：2026-06-27
- 相關：F-V1、F-V14、R13（[ROADMAP](../ROADMAP.md)）、[ADR-0003 相對強度σ](ADR-0003-relative-strength-normalization.md)、[ADR-0007 逆勢強勢](ADR-0007-counter-tide-strength.md)、[ADR-0008 中性門檻σ](ADR-0008-neutral-threshold-sigma.md)、[ADR-0009 逆勢強勢1σ門檻](ADR-0009-counter-tide-sigma-floor.md)

## 背景

目前座標有 2 種尺度（`S.norm`）：金額（絕對億）、相對強度（÷自身σ）。「逆勢強勢」（ADR-0007）只在退潮時用金色標出 x/σ 前 N 名，是**離散高亮、非連續座標**。需求是把「相對大盤的強弱」升級成**可連續比較、可排序**的第 3 種座標。

## 決定

新增第 3 種座標尺度「**市場相對**」。座標切換由 2 尺度擴為 3 尺度（金額／相對強度／市場相對）。

- **X（市場相對）** ＝ 各板塊自身σ強度 `x/σ` 扣掉「所有顯示板塊的平均σ強度」：
  `marketRelX_i = x_i/σ_i − mean_j(x_j/σ_j)`
  （`mean_j(x_j/σ_j)` ＝ 該分群當日的「大盤共同潮水」在σ空間的版本。）
- **Y、泡泡大小**：沿用相對強度模式（÷自身σ）。
- **金色逆勢強勢高亮**：保留為 top-5 標記，與新軸互補（[ADR-0009](ADR-0009-counter-tide-sigma-floor.md) 的 1σ 門檻續用）。
- 顏色/中性與象限：在市場相對模式下，以 `marketRelX`（demean 後的σ值）的正負與 1σ 帶判定（右＝高於大盤、左＝低於大盤；|marketRelX|<1 視為「與大盤同步」的中性灰）。

## 關鍵設計修正（資料探索發現）

原始直覺「`(x − 大盤潮水) ÷ σ`」會**爆掉**：以 70 日真實資料試，「板塊平均」≈6.4 億，對小板塊（造紙 σ≈0.5 億）是天文數字，造紙被甩到 **−11.5σ**，小板塊全擠到極端、圖壞掉。

**正解是換順序**：先把每個板塊換成σ強度 `x/σ`，**再**扣掉「大家的平均σ強度」。如此全部落在 O(1) 的σ單位、公平。退潮日 6/15 實測：數位雲端 +2.40σ（領先大盤）、半導體 −1.12σ（被賣超過同儕），範圍約 −1.2σ~+2.4σ，合理且直覺。

## 理由

- 補上「對照大盤」維度的完整連續版，把零散金色升級成可排序座標（延續 ADR-0007）。
- 在σ空間 demean，公平跨板塊（延續 ADR-0003／0008），且對「證交所產業」與「AI 主題」兩種分群都自洽（用該群自己的平均，而非全市場 tide）。

## 後果

- 好處：相對大盤的強弱可連續比較、排序。
- 代價／須知：
  - 第 3 個模式 ＝ 更多認知負擔（ADR-0003 已提醒）；切換與說明要做白話。
  - 實作牽涉把 `S.norm`（布林）擴為 3 態尺度，並讓 `mnorm`／`col`／`quad`／`counterTops`／座標標籤／資訊卡／localStorage 都支援；demean 需當日跨板塊平均，故渲染前先算一次。
- 連帶改：`index.html`、PRD（F-V14）、使用者說明、CHANGELOG、ROADMAP（R13 標完成）。

## 考慮過的替代方案

- **`(x − 大盤潮水絕對額) ÷ σ`**：小板塊爆表（−11σ），不採（即上述修正）。
- **只扣潮水、不除σ**：大型股偏誤重現（盤子大≠真的強），不採。
- **用實際大盤 tide ÷ 市場σ 來 demean**：對「AI 主題」分群不自洽（tide 是全市場、非該群），故改用該群的 cross-sectional 平均。
